put /urls/{urlID}/update
Update an existing shortened URL (updateShortenedURL)

Path parameters

urlID (required)
Path Parameter — The urlID to update

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body urlID_update_body (required)
Body Parameter —
example: { "value" : { "original_url" : "https://www.sierramadre.com/lettinggo" } }

Query parameters

Responses

200

Successfully updated the shortened URL

400

Invalid URL format

401

Unauthorized, authentication required

403

Forbidden, only admins can update

404

urlID not found

500

Server error

post /users/new
Create a new user (createUser)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body users_new_body (required)
Body Parameter —
example: { "value" : { "username" : "frankcatton", "email" : "fcatton@ocean.com", "password" : "securePassword123" } }

Return type

inline_response_201

Example data

Content-Type: application/json
{
  "user_id" : "user_id",
  "email" : "email",
  "username" : "username"
}

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

User successfully created inline_response_201

Example data

Content-Type: application/json
{"user_id":"1001","username":"frankcatton","email":"fcatton@ocean.com"}

400

Bad request (invalid input)

409

Conflict (username or email already exists)

500

Server error
